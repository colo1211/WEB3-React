=> 아 이거 너무 보기 더럽고 복잡하다고 생각할 수 있고 이것을 해결하기 위한 방안이 바로
'Component' 
 
[React Component 문법]
: HTML을 한 단어로 치환해서 쓸 수 있는 방법, 즉 'HTML 함수화'

방법
1. function APP (){} 이 끝나는 곳 바로 다음에 위치에 가서 작성한다. 즉, function App()과 나란히 작성한다. 이것도 Component 이기 때문이다.  
2. 함수 만들고 이름을 짓는다. (대문자로 시작해서 만든다. 소문자로 시작하면 렌더링 안됨)
3. 축약을 원하는 HTML 을 넣고
4. function APP(){} 내의 원하는 곳에서 <함수명> </함수명>을 작성한다. 

유의사항
1. 컴포넌트 작명의 시작은 무조건 대문자로 시작하여야 한다. 
2. return() 내의 태그는 무조건 태그 하나로 묶어야 한다. (위와 동일)
* fragment <> </> 를 사용하면 여러 개의 div를 사용 가능하다. 
<>
<div>
</div>
</>

무엇을 컴포넌트화 시키는게 좋은가? 
1. 반복 출현하는 HTML 덩어리들
2. 자주 변경되는 HTML UI들
3. 다른 페이지를 만들 때도 사용

단점 

state 를 쓸때 복잡해진다. => App 컴포넌트 내에서 선언했기 때문에 사용 불가
-> 상위 컴포넌트에서 만든 state를 사용하려면 props 문법을 사용해야 한다.
