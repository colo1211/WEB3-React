"컴포넌트를 3개 중첩해서 만들기"
-> state 전달이 복잡해진다.  
원시적인 방법으로 props를 여러번 전송한다. 

[재고 데이터 만들기]

재고량은 App.js(메인 컴포넌트) 에 state에 만든다. 
Detail.js(자식 컴포넌트)에서 쓰이더라도 데이터들을 위(부모)에서 아래(자식)로 흐르게 하는 것이 국룰이다. 

Detail.js (자식 컴포넌트)에서 
재고는 컴포넌트로 만들어 본다

⭐⭐⭐⭐⭐ 컴포넌트 안의 컴포넌트 안의 컴포넌트도 props를 통해서 state를 계속 전달 할 수 있다. 

정리 
1. 하위 컴포넌트가 몇개던 간에 데이터를 아래로 전송하기 위해서는 props를 쓴다. 
=> 이런 props 지옥을 막기 위해서 ⚠[useContext 혹은 Redux를 사용]한다. 

2. 하위 컴포넌트가 상위 컴포넌트를 변경하기 위해서는 state 변경함수를 똑같이 쓴다. 
변경함수 또한 props 로 전송한다.
